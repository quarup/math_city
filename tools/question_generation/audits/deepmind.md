# DeepMind `mathematics_dataset` — ingestion audit

This document is the input to the dataset-ingestion sub-track decisions for
DeepMind. It examines every submodule, samples real items, and assigns a
verdict per submodule:

- **variety** — the underlying math is already covered by our algorithmic
  generators; ingesting adds *phrasing variety* (reading-practice on top of
  arithmetic). Worth ingesting *if* the per-item filtering cost is low.
- **gap-fill** — the math isn't covered by any current generator. High value
  to ingest (subject to format complexity).
- **out_of_scope** — the math is outside CCSS K–8 (HS algebra, calculus,
  number bases, surds) OR too complex in framing for our K–8 audience.
- **skip** — within K–8 scope in principle but DeepMind's specific
  parametrisation produces items too noisy / too large / too obscure to be
  worth the filtering cost.

The raw sampled items live in [deepmind_samples.md](deepmind_samples.md),
auto-regenerated by [`audit_deepmind.py`](../audit_deepmind.py).

---

## Roll-up

**Worth ingesting (variety, filterable):** ~13 submodules.
**Worth ingesting (gap-fill candidates):** 2–4 submodules, needs runtime
work for non-integer answer formats.
**Out of scope / skip:** ~20 submodules.

Of DeepMind's ~39 K–8-eligible submodules, the realistically valuable
slice is ~15. After landing all of them we'd end up with maybe 2–3K
phrasing-varied items across G3–G8 arithmetic and number-theory
concepts, plus a handful of genuinely new (gap-fill) item shapes.

This is **modest pedagogical lift for substantial ingestion work**.
Recommendation: prioritise the gap-fill candidates and the highest-impact
variety submodules (those covering grades where our text-only generators
produce the most repetitive prompt shapes, like `place_value_*` and
`elapsed_time`); defer the rest indefinitely.

## Recommended ingestion order

If we decide to expand DeepMind ingestion beyond Chunk 80's `add_or_sub`,
this is the order that maximises (educational value / unit of filtering
work):

1. **`numbers.place_value`** — best fit, near-zero filtering, clean format.
2. **`measurement.time`** — true variety on elapsed-time; format aligns.
3. **`arithmetic.mul`** (whole-number subset only) — pairs with the
   existing `mult_facts_*` generators.
4. **`numbers.round_number`** — split into integer / decimal sub-buckets.
5. **`numbers.gcd` / `numbers.lcm`** — small operand range, clean phrasing.
6. **`numbers.is_prime` / `numbers.is_factor` / `numbers.list_prime_factors`** —
   factor-family triplet; ingest together.
7. **Gap-fill, runtime-blocked:** `comparison.{closest, kth_biggest, sort}`
   and `polynomials.evaluate` require the runtime to handle a couple of
   new answer-format shapes (letter-MC, comma-list, function notation).
   Worth a separate chunk once that runtime support lands.

Everything else (heavy filtering needed, or out of scope) is best left
unhauled.
